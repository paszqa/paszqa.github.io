Simple Monitoring Github Github link

A tool used to keep track of current and historical state of hardware, services and processes. Small scale, simple and performant. Made for personal use to monitor Raspberry Pi at home.

The project is kept within a single shell script, which consists of multiple checks - ex. CPU, filesystem, process. User chooses which check to run via command line parameter and it generates a dedicated CSV file with semicolon separated date + results.

  • Bash scripting
  • Linux administration
  • Automation

Cryptocurrency Tracker Github Github link

A tracker for chosen cryptocurrencies. Saves and generates multiple reports, based on CoinMarketCap website. It is fairly generic, and allows users to customize queries and reports according to their own preferences.

  • Python scripting
  • Data reporting

Get raw data

Running crypto.py queries the API of CoinMarketCap website and saves raw results for the currencies defined in the configuration.

Calculate according to user's input

Script calculate.py uses this data and values from user's portfolio file (which contains quantity of each coin and price at the time of purchase) to generate profit/loss CSV report.

The output

Running both scripts results in creation of 4 logfiles: a small file with newest prices, a large file with full historical data, and 2 reports with calculated values - small file with current profit for each cryptocurrency, and a large file with all previous general gain/loss values.

SteamtrackerGithub Github linkLinkLive example

A relatively complex tool, which tracks users game activity on Steam platform. It is designed to be launched once every 24 hours and (depending on account's library size) might need quite a while to complete the scan and generate a website.

  • Bash scripting
  • HTML
  • Data processing
  • CSS3
  • Automation

Adding a new user to track

Running addnewid.py creates a directory and a basic information file.

Getting website data into DB

Next step is executing site2DB.sh - using API key, all available historical data of the profile with chosen ID is scanned and moved to local database. If the user already exists in the DB, most recent values are compared to the previous entries and only delta is added to database, as it saves A LOT of storage space.

Monthly stats

Executing calcMonthFromDB.sh calculates the sum of time played for each month available, fills in the blanks and creates TXT/CSV files for quick access to the top 5 titles.

Generating the website

Script genWebSmallFromDB.sh accesses the database and relevant files and builds a static website with a table containing human-readable information - by default it includes the last 40 days.

Further development will consist of making the code more generic, improving the visual aspect of the website, adding interactivity and improving performance (ex. current sorting solution might freeze the browser for 5-15 seconds with as few as 30 rows). qBot for Discord Github Github link

An application / automation system for Discord (a pretty popular messaging software, mostly gaming-oriented), capable of delivering stats, information, trivia and any random stuff, as well as heavy data collection. Work in progress, but it's been running on a friendly server since 2021 with almost no issues.

  • NodeJS scripting
  • Python scripting
  • Data reporting
  • Automation
  • MariaDB

The application runs on a NodeJS server and is heavily based on extended versions of other scripts. qBot reacts to scheduled jobs and Discord users' actions - ex. certain messages, commands, or reactions - and acts accordingly. Each feature is a separate module with its own requirements specified.

It requires custom configuration per server, and keeps most of the data within a MariaDB database, with select chosen outputs saved within the filesystem to prevent querying the DB server and waiting for its response. It was designed for RaspberryPi, so limited computing power was always a concern.

One of the features is "Game Price" module, which allows users to type in a command and get current price of the game, its availability, and access options within seconds - plus it provides a link to a price aggregator, which offers even more information. The module acts like a user agent, and relies on scanning the website itself.

Another module, "Vote System" allows admin to specify vote options and a dedicated voting channel, prepares the posts, and tracks users' reactions. It only takes into the account the most recent vote of each user and pushes it to the database. In case of losing connectivity or failure, the module is able to rescan all of its posts and recreate any missing options, so no data is lost.

Next feature is data collection functionality, called "Activity". Bot runs a check on all users on predefined Discord server, and saves their name, ID, status, current activity, game title, Twitch ID and activity, as well as server's user count. Users are able to add their Steam accounts to the database manually, which are then managed by Steam Tracker. It runs every couple of minutes.

Data collected by other modules is used to generate an infographic of sorts - a summary of user activity on the server within the last 7 days. It queries the database filtering most recent data of each user, sums it up, creates CSV lists sorted by values, queries IGDB online API for game covers, downloads these covers, queries Twitch/Steam/Discord APIs for users' basic info and avatars, saves them and creates the image according to preset configuration.

Application logs all of its actions with entries properly tagged and timestamped, to make troubleshooting easier. Especially that qBot2 is still a work in progress project.
